# Configuration

## Core application

| Variable | Description |
|----------|-------------|
| `APP_NAME`, `APP_DESCRIPTION`, `APP_VERSION` | OpenAPI / UI metadata |
| `CONTACT_NAME`, `CONTACT_EMAIL`, `LICENSE_NAME` | API info block |
| `ENVIRONMENT` | `local`, `staging`, or `production`. Controls docs visibility and `/sources` UI |
| `LOG_VERBOSITY` | `full`, `medium`, or `minimal` |

## Database

| Variable | Local compose value |
|----------|---------------------|
| `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB` | From wizard |
| `POSTGRES_SERVER` | `db` |
| `POSTGRES_PORT` | `5432` |

## Redis

| Variable | Purpose |
|----------|---------|
| `REDIS_CACHE_HOST`, `REDIS_CACHE_PORT` | Client-side cache |
| `REDIS_QUEUE_HOST`, `REDIS_QUEUE_PORT` | ARQ worker queue |
| `REDIS_RATE_LIMIT_HOST`, `REDIS_RATE_LIMIT_PORT` | Rate limiting |

## Authentication

| Variable | Default |
|----------|---------|
| `SECRET_KEY` | Generated by wizard |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | `60` |
| `REFRESH_TOKEN_EXPIRE_DAYS` | `7` |

## Archive (CASDA)

| Variable | Purpose |
|----------|---------|
| `CASDA_USERNAME`, `CASDA_PASSWORD` | TAP / staging access |
| `CASDA_STAGE_BY_SBID` | Stage by scheduling block ID (default `true`) |

## Workflow automation & shaping

| Variable | Purpose |
|----------|---------|
| `WORKFLOW_EXECUTION_AUTOMATION_ENABLED` | Auto-schedule executions when sources are pending |
| `SHAPING_EXECUTION_MAX_IN_FLIGHT_RUNS` | Cap concurrent automated runs |
| `SHAPING_DISCOVERY_MAX_IN_FLIGHT_BATCHES` | Cap concurrent discovery batches |
| `SHAPING_ARQ_QUEUE_MAX_DEPTH` | Max ARQ queue depth |

## Discovery tuning

| Variable | Purpose |
|----------|---------|
| `DISCOVERY_BATCH_SIZE` | Sources per discovery batch |
| `DISCOVERY_SCHEDULE_MINUTES` | Cron interval for scheduled discovery |
| `DISCOVERY_STALE_HOURS` | Re-check sources after N hours |
| `DISCOVERY_CLAIM_TTL_MINUTES` | Discovery lease expiry |

## Workflow engines

| Variable | Values | Purpose |
|----------|--------|---------|
| `WORKFLOW_ENGINE_DISCOVERY` | `arq` or `restate` | Discovery batch backend |
| `WORKFLOW_ENGINE_EXECUTION` | `arq` or `restate` | Execution batch backend |
| `RESTATE_INGRESS_BASE_URL` | e.g. `http://restate:8080` | Restate HTTP ingress (compose service name) |

Local compose defaults both engines to **restate**.

## Slurm SSH (production/custom)

| Variable | Purpose |
|----------|---------|
| `SLURM_SSH_KNOWN_HOSTS_SOURCE` | Host path ⮕ `./deploy/ssh/known_hosts` |
| `SLURM_SSH_PRIVATE_KEY_FILE` | Host path ⮕ `./deploy/ssh/id_slurm` |

Compose mounts these into worker containers for `slurm_remote` deployment profiles.

## Interactive API docs by environment

| Environment | `/docs`, `/redoc`, `/openapi.json` |
|-------------|--------------------------------------|
| **local** | Public |
| **production** | Public |
| **staging** | Superuser bearer token required |

Authenticate with `POST /api/v1/login` (form-urlencoded username/password) to obtain a bearer token. Staging may require a superuser token for `/docs` and `/redoc`.
